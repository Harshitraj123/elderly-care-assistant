[1:00 — ARCHITECTURE]
(show assets/architecture_diagram.png)
Let's walk through how this works under the hood. 
Every user request starts at the Security Checkpoint Node, which handles PII scrubbing and prompt injection checks before forwarding it downstream.
Once cleared, the request reaches the Care Coordinator Orchestrator, which uses AgentTools to delegate tasks to either the Medication Manager or the Appointment Scheduler specialized sub-agents.
For sensitive actions like booking visits or altering schedules, the flow routes to the Human Approval Node to verify caregiver consent.
Approved actions proceed to the Execute Action Node, where the specialized sub-agents query the Stdio MCP Server tools to list medications, log pill compliance, or book calendars, before delivering the final outcome via the Final Output Node.
